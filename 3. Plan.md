## 🧩 Project Overview

**EquiSplit** is a QR-based group expense management web application built as a Final Year Project for a Bachelor of Information Systems student at UiTM Malaysia.

---

## 🛠️ Tech Stack

| Layer | Technology |
|---|---|
| **Backend** | CakePHP 5.x |
| **Frontend** | Tailwind CSS |
| **Database** | MySQL via Laragon |
| **Authentication** | Google OAuth 2.0 |
| **File Storage** | Cloudinary API |
| **Local Dev URL** | http://localhost/EquiSplit |
| **Hosting** | InfinityFree |

---

## 🗄️ Database — 9 Tables

```
users, group_sessions, participants, session_charges,
expenses, expense_allocations, payment_proofs,
settlement_transactions, session_notifications
```

---

## 👤 User Roles

| Role | Description |
|---|---|
| **Host** | Creates and manages sessions |
| **Registered Participant** | Joins via QR with a Google account |
| **Guest** | Joins via QR with name only — no session history |

---

## 🎛️ Session Presets

| Preset | Behaviour |
|---|---|
| **Food & Dining** | SST%, proportional split |
| **Road Trip** | Flat amounts, equal split |
| **Long Trip** | Collaborative ledger, net-debt settlement |
| **Custom** | Manually toggle all rules |

---

## 🧮 Calculation Logic

**Proportional Split:**
```
Total_i = Subtotal_i + (Subtotal_i × Tax%)
                     + (Subtotal_i / Group Subtotal × Flat Fees)
```

**Net-Debt Settlement:**
```
Balance_i = Total Paid − Total Owed  →  minimize transactions
```

---

## ⭐ Key Features

- UUID-based QR session URLs (no guessable IDs)
- Dynamic preset engine reconfigures UI and logic
- Mandatory payment proof upload via Cloudinary
- Host approves or rejects payment proofs
- Session-scoped receipt privacy
- Unpaid balances marked as **Unresolved** on session close
- Net-debt settlement engine runs on clean close
